## نظرة عامة

نظام إدارة الوقود والزيوت ده تطبيق full-stack اتعمل عشان يدير:
- **عمليات الوقود**: متابعة إمدادات الوقود، توزيعها على المحطات، الاستهلاك، وقياسات الخزانات
- **مخزون الزيوت**: إدارة إمدادات الزيوت، تخصيصها للمستهلكين، معدلات الاستهلاك، وعمليات الصرف
- **البنية التحتية**: إدارة المحطات، الخزانات، أنواع الوقود، ومنتجات الزيوت
- **المستخدمين والأمان**: صلاحيات حسب الدور (أدمن ومستخدم عادي)
- **سجل المتابعة (Audit Trail)**: تسجيل كامل لكل حركة بتحصل في النظام

التطبيق ده متبني بأحدث تقنيات الويب وأفضل الممارسات عشان يبقى قابل للتوسع، آمن، وسهل الصيانة.

## أهم المميزات

### 🔐 تسجيل الدخول والصلاحيات
- تسجيل دخول باسم مستخدم وباسورد بس (من غير تسجيل دخول عن طريق حسابات سوشيال)
- صلاحيات حسب الدور:
  - **superadmin** — صلاحية كاملة على النظام
  - **admin** — صلاحيات الإدارة والتشغيل
  - **user** — صلاحيات تشغيلية محدودة
- إدارة الجلسات مع تتبع الـ IP والـ user agent
- متابعة حالة الحساب (نشط/غير نشط)
- إعداد أدمن افتراضي عشان بداية التشغيل (bootstrap)

### ⛽ إدارة الوقود
- **إدارة أنواع الوقود** — ضبط أنواع الوقود (ديزل، بنزين 80، بنزين 92، إلخ) مع معدلات التحويل (طن لللتر)
- **المحطات والخزانات** — إدارة هرمية للمحطات، وكل محطة فيها أكتر من خزان
- **مواصفات الخزانات** — متابعة السعة (بالطن/باللتر)، الرصيد الحالي، وحدود التنبيه
- **إمدادات الوقود (الوارد)** — تسجيل شحنات الوقود بـ:
  - تتبع المستندات والفواتير
  - بيانات المورد
  - تتبع السعر (وحدة وإجمالي)
  - التوزيع على أكتر من محطة وخزان
  - ترقيم الوارد لكل محطة
- **استهلاك الوقود** — تسجيل الاستهلاك اليومي لكل خزان مع ملاحظات وتتبع المستخدم
- **قياسات الخزانات** — تسجيل القياسات الفعلية مقابل الرصيد النظري مع تحليل الفروقات
- **كشوف الرصيد اليومية** — تسوية تلقائية للرصيد الافتتاحي، الإمدادات، الاستهلاك، والرصيد الختامي مع اكتشاف أي فروقات

### 🛢️ إدارة الزيوت
- **كتالوج منتجات الزيوت** — مخزون شامل للزيوت بـ:
  - أنواع وحدات مختلفة (زجاجة، لتر، كيلو، كرتونة، برميل)
  - تسعير لكل وحدة
  - وحدات تجميعية (براميل، جراكن، إلخ) مع تحديد الكمية
  - حدود تنبيه ومتابعة الرصيد الحالي
- **إمدادات الزيوت (الوارد)** — متابعة شحنات الزيوت الواردة:
  - تتبع المورد والعقد
  - إدارة الفواتير والتواريخ
  - تتبع التكلفة والملاحظات
- **المستهلكين** — إدارة الجهات المستقبلة (ورش الصيانة، مراكز الخدمة، الوحدات الفنية)
- **معدلات استهلاك الزيوت** — تحديد معدلات استهلاك دورية لكل مستهلك:
  - ضبط المعدل لكل مستهلك ولكل نوع زيت
  - متابعة الفترة (أسبوعي أو شهري)
  - جدولة تلقائية لتاريخ إعادة التعبئة
- **معاملات الزيوت (الصادر)** — متابعة صرف الزيوت بـ:
  - توليد رقم تسلسلي
  - بيانات المستلم (الاسم والرتبة)
  - تتبع الشخص المسؤول عن الصرف
  - ملاحظات ومتابعة كاملة للمعاملة

### 📊 التقارير والتحليلات
- **نظرة عامة على لوحة التحكم** — عرض أهم المؤشرات والتنبيهات
- **عرض الأرصدة اليومية** — الاطلاع على بيانات التسوية على مستوى المحطات وأنواع الوقود
- **نظام التنبيهات** — إشعارات مهمة عن:
  - انخفاض مستويات الوقود/الزيت
  - فروقات في قياسات الخزانات
  - حالات استهلاك غير طبيعية
  - مستويات خطورة متعددة (منخفض، متوسط، عالي، حرج)
- **سجل المتابعة (Audit Log)** — تتبع كل عمليات النظام مع بيانات المستخدم

### 👥 إدارة المستخدمين
- إنشاء وإدارة مستخدمي النظام مع تحديد الأدوار
- متابعة نشاط المستخدم في سجلات المتابعة
- تفعيل/تعطيل الحسابات
- بيانات البروفايل (الاسم، الإيميل، رقم التليفون، اسم المستخدم المعروض)

### ⚙️ إدارة النظام
- **إدارة الإعدادات** — تخزين إعدادات النظام بنظام key-value
- **سجل المتابعة (Audit Trail)** — تسجيل شامل لـ:
  - أفعال المستخدمين (إنشاء، تعديل، حذف، دخول، خروج)
  - لقطات البيانات قبل وبعد التعديل
  - التوقيت وبيانات المستخدم
- **دعم المرفقات** — رفع ملفات مرتبطة بسجلات مختلفة

## تسجيل الدخول والصلاحيات

### نظرة عامة
التطبيق بيستخدم **Better Auth** لتسجيل دخول آمن وإدارة جلسات، مربوط بتخزين PostgreSQL.

### طريقة تسجيل الدخول
- **اسم مستخدم وباسورد بس** — مفيش تسجيل دخول عن طريق حسابات سوشيال أو OAuth
- الباسوردات متخزنة بأمان عن طريق تشفير Better Auth
- توكنز الجلسات متخزنة في جدول `session` مع تاريخ انتهاء

### أدوار الصلاحيات
```
┌─────────────┬──────────────────────────────────────────────┐
│ الدور        │ الصلاحيات                                     │
├─────────────┼──────────────────────────────────────────────┤
│ superadmin  │ صلاحية كاملة على النظام (محجوزة للمستقبل)      │
│ admin       │ إدارة كل العمليات، المستخدمين، والإعدادات      │
│ user        │ صلاحيات تشغيلية محدودة                         │
└─────────────┴──────────────────────────────────────────────┘
```

### حساب الأدمن الافتراضي
عند أول تشغيل، اعمل تهيئة لمستخدم الأدمن:

**النقطة:** `GET /api/setup`
- بتشتغل بس لما قاعدة البيانات تكون فاضية
- بتنشئ مستخدم أدمن افتراضي

**بيانات الدخول الافتراضية:**
- اسم المستخدم: `admin`
- الباسورد: `adminadmin123`
- الإيميل: `admin@transport.gov.eg`

**تغيير باسورد التهيئة الافتراضي:**
حدد متغير البيئة `SETUP_ADMIN_PASSWORD` في `.env.local`

### إدارة الجلسات
- الجلسات فيها بيانات الـ IP والـ user agent للأمان
- انتهاء تلقائي عن طريق `expiresAt`
- عزل الجلسات حسب كل مستخدم

## الإعداد والتثبيت

### المتطلبات
- Node.js 18+ و pnpm
- قاعدة بيانات PostgreSQL 12+
- ملف `.env.local` معمول (شوف تحت)

### إعداد متغيرات البيئة

اعمل ملف `.env.local` في المجلد الرئيسي:

```bash
# قاعدة البيانات
DATABASE_URL=postgres://username:password@localhost:5432/fuel_management

# المصادقة
BETTER_AUTH_URL=http://localhost:3000
BETTER_AUTH_SECRET=your-secret-key-here

# اختياري: تغيير باسورد تهيئة الأدمن
SETUP_ADMIN_PASSWORD=your-custom-admin-password

# بيئة Node
NODE_ENV=development
```

**صيغة رابط قاعدة البيانات:**
```
postgres://[user[:password]@][netloc][:port][/dbname]
```

### إعداد قاعدة البيانات

1. **اعمل قاعدة بيانات PostgreSQL:**
   ```bash
   createdb fuel_management
   ```

2. **شغّل الـ migrations** (لو بتستخدم Drizzle migrations):
   ```bash
   # الـ migrations المفروض تكون في db/migrations/ أو حاجة شبهها
   # اتبع دليل Drizzle ORM migration
   ```

3. **تهيئة الهيكل (Schema):**
   - الهيكل معرّف في `lib/db/schema.ts`
   - استخدم `db push` بتاع Drizzle للتطوير
   - أو اعمل الجداول يدويًا حسب تعريفات الهيكل

## التشغيل محليًا

### 1. تثبيت الاعتماديات
```bash
pnpm install
```

### 2. إعداد البيئة
```bash
cp .env.example .env.local
# عدّل .env.local بإعدادات قاعدة البيانات والمصادقة بتاعتك
```

### 3. إعداد قاعدة البيانات
```bash
# اعمل جداول قاعدة البيانات (وضع تطوير Drizzle)
pnpm drizzle-kit push:pg
```

### 4. شغّل سيرفر التطوير
```bash
pnpm dev
```

التطبيق هيبقى متاح على `http://localhost:3000`

### 5. تهيئة مستخدم الأدمن
افتح `http://localhost:3000/api/setup` في المتصفح (مرة واحدة بس)

### 6. تسجيل الدخول
روح لـ `http://localhost:3000/sign-in` واستخدم بيانات الأدمن الافتراضية

## مسارات الـ API

### المصادقة
- `POST /api/auth/sign-in` — تسجيل دخول المستخدم
- `POST /api/auth/sign-out` — تسجيل خروج المستخدم
- `GET /api/auth/session` — جلب الجلسة الحالية
- `POST /api/auth/sign-up` — تسجيل مستخدم جديد (لو مفعّل)

### الإعداد
- `GET /api/setup` — تهيئة مستخدم الأدمن (بتشتغل مرة واحدة لما قاعدة البيانات تكون فاضية)

### عمليات الوقود
- `POST /api/fuel-supplies` — إنشاء سجل إمداد وقود
- `GET /api/fuel-supplies` — عرض قائمة إمدادات الوقود
- `POST /api/fuel-supplies/[id]` — تعديل إمداد وقود
- `DELETE /api/fuel-supplies/[id]` — حذف إمداد وقود
- `GET /api/fuel-supplies/next-import-number` — جلب رقم الوارد التالي

### عمليات الزيوت
- `POST /api/oil-supplies` — إنشاء سجل إمداد زيت
- `GET /api/oil-supplies` — عرض قائمة إمدادات الزيوت

### الترحيل (Migrations)
- `POST /api/migrate` — تشغيل ترحيل قاعدة البيانات أو استيراد البيانات

## حل المشاكل

### مشاكل الاتصال بقاعدة البيانات
- تأكد من `DATABASE_URL` في `.env.local`
- تأكد إن PostgreSQL شغال
- افحص بيانات دخول قاعدة البيانات
- تأكد إن الـ firewall بيسمح بالاتصالات

### إعداد الأدمن مش شغال
- قاعدة البيانات لازم تكون فاضية (مفيش سجلات في جدول `user`)
- افحص اتصال قاعدة البيانات الأول
- تأكد إن `BETTER_AUTH_SECRET` متحدد
- افحص لوجز السيرفر عشان تشوف تفاصيل الخطأ

### أخطاء البناء (Build)
- امسح مجلد `.next`: `rm -rf .next`
- أعد تثبيت الاعتماديات: `rm -rf node_modules pnpm-lock.yaml && pnpm install`
- تأكد من أخطاء TypeScript: `pnpm tsc --noEmit`
